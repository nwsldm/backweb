## How Deployments Work

DeployHub follows a simple, automated deployment workflow:

### 1. Push to Git

When you push code to your connected repository, we automatically detect the change.

### 2. Build Process

We clone your repository and build your application using the appropriate build command for your framework:

- **Next.js**: `npm run build`
- **Nuxt**: `npm run build`
- **React/Vue/Svelte**: `npm run build`
- **Static Sites**: No build needed, we serve files directly

### 3. Deploy to Edge

After building, we distribute your application to our global edge network of 100+ locations.

### 4. Preview Deployments

Every pull request automatically gets a preview deployment URL so you can test changes before merging.

## Configuration

You can configure your deployment in the project settings:

- **Build Command**: Customize the build command
- **Environment Variables**: Add secrets and configuration
- **Branch Settings**: Configure which branches trigger deployments
- **Custom Domains**: Add your own domain names

## Rollbacks

If something goes wrong, you can rollback to any previous deployment in one click. We keep a history of all deployments with timestamps and commit information.
