## Setting Environment Variables

Environment variables let you configure your application without hardcoding values.

### Adding Variables

1. Go to your project settings
2. Click "Environment Variables"
3. Enter a key and value
4. Click "Add Variable"

### Types

- **Production Variables**: Available in production deployments
- **Preview Variables**: Available in preview deployments (pull requests)
- **Secret Variables**: Values are encrypted and not visible once saved

### Security

All environment variables are encrypted at rest and in transit. Secret variables are never logged or displayed in the dashboard.

### Accessing Variables

In your application code, access variables using standard environment variable syntax:

```javascript
const apiUrl = process.env.API_URL
const apiKey = process.env.API_KEY
```

### Framework Specifics

**Next.js**: Variables prefixed with `NEXT_PUBLIC_` are available in the browser.

**Nuxt**: Use `useRuntimeConfig()` to access variables.

**Vite**: Variables prefixed with `VITE_` are available in the browser.
